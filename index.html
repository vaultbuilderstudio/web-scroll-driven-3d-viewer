Scroll-driven 3D Model Section

A pinned, scroll-linked 3D viewer built with Three.js + GSAP ScrollTrigger. The model view is deterministic. As you scroll, the camera moves from a defined START pose to a defined END pose.

This is a standalone demo of the same interaction pattern we can use across homepage sections and future product configuration experiences.

Scroll controls the camera

This section pins during scroll and maps scroll progress (0 â†’ 1) to a camera move across the model. We capture exact START and END poses, then interpolate position, rotation, target, and dolly distance. This creates a stable, repeatable, and easy to tune experience.

  • Deterministic: camera is driven only by scroll
  • Tunable: START and END poses can be updated for composition
  • Reusable: the same pattern supports feature reveals and config-driven states

A reusable interaction pattern

This workflow is valuable prep for Neo-config and future vehicle models. The requirements for real-time web use stay the same no matter how the 3D model is driven. The GLB must be built with predictable hierarchy, clean materials, and solid optimization so it can be controlled deterministically. In this demo, the input is scroll progress. In the configurator, the input becomes configuration state, driving visibility, materials, and animation hooks reliably.
